Key API Endpoints

The Doc-Assist-Pro system exposes the following key API endpoints for interactions between frontend applications and the backend:

Authentication Endpoints

Method Endpoint Description Auth Required
POST /api/auth/register Register a new user No
POST /api/auth/login Authenticate and get token No
GET /api/auth/me Get current user profile Yes
POST /api/auth/logout Logout user and invalidate token Yes

Patient Endpoints

Method Endpoint Description Auth Required
GET /api/patients/profile Get patient profile Yes (Patient)
PUT /api/patients/profile Update patient profile Yes (Patient)
GET /api/patients/dashboard Get patient dashboard data Yes (Patient)
GET /api/patients/medical-records Get patient medical records Yes (Patient)

Doctor Endpoints

Method Endpoint Description Auth Required
POST /api/doctors/register Register as a doctor No
GET /api/doctors/profile/me Get doctor profile Yes (Doctor)
PUT /api/doctors/profile/me Update doctor profile Yes (Doctor)
GET /api/doctors/dashboard Get doctor dashboard data Yes (Doctor)
GET /api/doctors/patients Get doctor's patients Yes (Doctor)

Appointment Endpoints

Method Endpoint Description Auth Required
GET /api/appointments/my-appointments Get user's appointments Yes
GET /api/appointments/:id Get appointment details Yes
POST /api/appointments Create new appointment Yes (Patient)
PUT /api/appointments/:id Update appointment Yes
POST /api/appointments/:id/cancel Cancel appointment Yes
GET /api/appointments/today Get today's appointments Yes (Doctor)
GET /api/appointments/doctors/:id/availability Get doctor availability Yes

Consultation Endpoints

Method Endpoint Description Auth Required
POST /api/consultations/appointment/:id/start Start consultation for appointment Yes (Doctor)
GET /api/consultations/:id Get consultation details Yes
POST /api/consultations/:id/complete Complete consultation Yes (Doctor)
POST /api/consultations/:id/missed Mark consultation as missed Yes (Doctor)
POST /api/consultations/:id/medical-record Add medical record to consultation Yes (Doctor)
POST /api/consultations/:id/prescription Add prescription to consultation Yes (Doctor)
POST /api/consultations/:id/submit Submit complete consultation data Yes (Doctor)

Feedback Endpoints

Method Endpoint Description Auth Required
POST /api/feedback/appointment/:id Submit feedback for appointment Yes (Patient)
GET /api/feedback/appointment/:id Get feedback for appointment Yes
GET /api/feedback/doctor/:id Get all feedback for doctor Yes

Health Tips Endpoints

Method Endpoint Description Auth Required
GET /api/health-tips Get health tips Yes
GET /api/health-tips/personalized Get personalized health tips Yes
GET /api/health-tips/:id Get specific health tip Yes

Admin Endpoints

Method Endpoint Description Auth Required
GET /api/admin/dashboard Get admin dashboard overview Yes (Admin)
GET /api/admin/doctors Get all doctors (including pending verification) Yes (Admin)
GET /api/admin/doctors/:id Get doctor details Yes (Admin)
POST /api/admin/doctors/:id/verify Approve and verify a doctor's account Yes (Admin)
POST /api/admin/doctors/:id/reject Reject a doctor's verification request Yes (Admin)
GET /api/admin/patients Get all patient accounts Yes (Admin)
GET /api/admin/patients/:id Get patient details Yes (Admin)
PUT /api/admin/patients/:id/status Update patient account status (active/inactive) Yes (Admin)
PUT /api/admin/doctors/:id/status Update doctor account status (active/inactive) Yes (Admin)
GET /api/admin/statistics Get system statistics and metrics Yes (Admin)
GET /api/admin/activity-logs Get system activity logs Yes (Admin)
